## 背景

阶段 E 引入后台运行、取消、上传校验和工作台历史。复核发现这些功能的接口表面存在，但运行中的取消、文件内容校验和长耗时任务的工作台交互未满足预期。

## 用户可见行为

1. `POST .../execute` 返回 `202 Accepted`，耗时管道不占用 API 事件循环；进度和取消接口仍可调用。
2. 用户在某个管道步骤执行期间请求取消后，下一步骤开始前运行会变为 `cancelled`，且不会覆盖已持久化的取消标记。
3. 把 PDF 等二进制文件伪装成 `.txt` / `.md` 上传会被拒绝，返回 `FILE_MIME_MISMATCH`。
4. 工作台只在运行完成后请求结果产物；失败或取消时明确显示无可下载产物，而不是返回误导性的 404。

## 非目标

- 不强制中断正在执行的单次模型调用；取消是步骤边界的协作式取消。
- 不在本次加入用户认证或跨项目授权；该能力属于路线图后续权限阶段。

## 验收

- [x] 后台执行转入工作线程，ASGI 事件循环可继续处理请求。
- [x] 外部取消信号在下一步骤前生效。
- [x] 扩展名与内容签名不一致时拒绝上传。
- [x] 工作台长运行轮询终态，并按终态处理产物。
